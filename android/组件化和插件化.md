# 组件化
>完成一组特定功能的程序，可以是源码级别的，也可以是运行时加载的。

分类：
    库组件
    公共组件（登陆、支付、消息推送，购物车）
    业务组件（超市、秒杀、服饰、生鲜等）

解决问题：
    大型项目协作开发（操作系统、微信）
    强制解耦
    便于项目拆分

引入问题：
    组件对运行环境不是完全已知的。
    组件和运行环境版本冲突

    依赖问题（组件之间依赖、第三方依赖）
    相互调用（同步、异步）
    相互通信

    ARouter 将页面抽象成链接。
    EventBus 组件之间通信（异步）
    接口隔离  

基础问题
    组件的拆分和整合
    组件之间的通信

# 插件化
    接口
    加载，卸载（配置驱动）

>插件(Plug-in,又称addin、add-in、addon或add-on,又译外挂)是一种遵循一定规范的应用程序接口编写出来的程序。其只能运行在程序规定的系统平台下（可能同时支持多个平台），而不能脱离指定的平台单独运行。
